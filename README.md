# mm-dlp: Next-Generation, High-Performance Async Rust Media Downloader

**mm-dlp** is a next-generation media downloader engineered from the ground up in asynchronous Rust. Our primary objective is to create the definitive open-source community standard for extracting and downloading media from any online platform, surpassing the capabilities of popular tools like `yt-dlp` in performance, security, and cross-platform embeddability.

This project is built on a foundation of a high-performance, async-first Rust core, a sandboxed plugin system for site extractors, and a unified Foreign Function Interface (FFI) for seamless integration with modern user interfaces across desktop and mobile.

## Core Features

*   **High-Performance Async Core:** Built on `tokio`, our engine leverages zero-cost async tasks to handle thousands of concurrent operations, maximizing network and disk I/O without the overhead of traditional threading models.
*   **Advanced Network Impersonation:** A sophisticated network engine designed to bypass modern WAFs (like Cloudflare and Akamai) by precisely mimicking browser TLS/JA3/JA4 and HTTP/2 fingerprints.
*   **Sandboxed Extractor Engine:** Site-specific logic runs in a secure, isolated sandbox using WebAssembly (WASM) or a high-speed JavaScript runtime (QuickJS), preventing security vulnerabilities and ensuring stable performance.
*   **Out-of-Order Segment Downloading:** A highly efficient downloader that fetches HLS/DASH segments in parallel and reassembles them sequentially, ensuring maximum bandwidth saturation and minimal memory usage.
*   **Cross-Platform FFI:** A clean, unified interface generated by `uniffi` allows `mm-dlp-core` to be embedded as a high-performance library in any application, with first-class support for Kotlin Multiplatform (Android/Desktop), Swift (iOS), and Dart (Flutter).
*   **Memory-Safe by Design:** By leveraging Rust's ownership model and zero-copy buffer management, we eliminate entire classes of bugs, memory leaks, and performance bottlenecks common in garbage-collected languages.

## Multi-Language FFI Bindings

Thanks to `uniffi` and WebAssembly, `mm-dlp-core` can be embedded almost anywhere. We provide first-class support for the following platforms, with auto-generated, type-safe, and idiomatic bindings:

*   **C/C++:** Generate standard C headers for low-level embedding in any native application.
*   **Python:** Idiomatic bindings for scripting and integration with Python applications.
*   **JavaScript & TypeScript (Node.js/React/jQuery):** The core library can be compiled to WebAssembly (WASM) for use in web browsers, Node.js backends, and frameworks like React or jQuery.
*   **Kotlin & Java:** First-class support for integration into modern Android and desktop/JVM applications.
*   **C# (.NET):** Bindings for the .NET ecosystem.
*   **Dart (Flutter):** High-performance bindings for cross-platform Flutter apps.

## Why Rust over Python?

While `yt-dlp` is an incredible tool, its Python foundation presents inherent limitations that Rust is uniquely positioned to solve:

1.  **The Global Interpreter Lock (GIL):** Python's GIL prevents true parallelism for CPU-bound tasks. `mm-dlp` uses `tokio` to run thousands of concurrent async tasks on a fixed pool of OS threads, achieving superior performance for I/O-bound work like network requests and disk writes.
2.  **High Memory Overhead:** Python's object model and garbage collector lead to significant memory usage and unpredictable latency spikes, especially when handling large files or many small data chunks. Rust's zero-copy data handling and compile-time memory management keep memory usage flat and predictable.
3.  **Slow JavaScript Interpretation:** Modern websites use obfuscated JavaScript to protect their media streams. `yt-dlp`'s Python-based JS interpreter is a major bottleneck. `mm-dlp` embeds a highly optimized, JIT-compiled JavaScript engine (QuickJS), executing these ciphers at near-native speed.

## Building from Source

This project is a standard Cargo workspace.

```bash
# Clone the repository
git clone https://github.com/your-username/mm-dlp.git
cd mm-dlp

# Build the entire workspace in release mode
cargo build --release

# Run the CLI
./target/release/mm-dlp-cli --url "https://example.com/video"
```
